# CarDet 修改说明

## 修改概览

本次在保留 Qt/C++ 桌面程序结构的基础上，补齐课程验收中的数据集管理、标注、训练指标入库、数据库表、量化入口、推理降级方案、菜单工具栏和结果可视化。

主要修改文件：

- `CarDet.pro`：加入新增 C++ 文件和 `scripts/quantize.py`，构建标准切换为 C++20。
- `src/model/DataTypes.h`：训练参数增加训练集、验证集和输出目录。
- `src/model/DatabaseManager.h/.cpp`：新增 `datasets`、`images`、`categories`、`annotations`、`training_runs`、`model_files`、`inference_results` 等价表和写入接口。
- `src/model/QuantizedOnnxRunner.h/.cpp`：新增 C++ ONNX Runtime 条件编译推理接口；未配置 ORT 时给出明确提示并允许 GUI 回退到 Python 推理。
- `src/MainWindow.h/.cpp`：增加菜单栏、工具栏、状态栏和页面快捷切换。
- `src/view/DatasetPage.h/.cpp`：支持创建 YOLO 数据集、打开数据集、导入图片文件夹、展示图像与标注状态。
- `src/view/AnnotationCanvas.h/.cpp`：支持画框、删除、移动、四角缩放、平移、旋转、撤销、重做、选中框类别更新。
- `src/view/AnnotationPage.h/.cpp`：新增类别列表、新增类别、删除类别、应用类别到选中框、可编辑标注表，保存时写入 SQLite。
- `src/view/TrainingPage.h/.cpp`：新增训练集 images、验证集 images、输出目录控件；训练结束后显示结构化指标并入库。
- `src/viewmodel/TrainingViewModel.h/.cpp`：解析 `BEST_MODEL:` 和 `FINAL_RESULT:`，把指标 JSON 传回 GUI。
- `src/view/DeployPage.cpp`：量化入口改为调用统一脚本 `scripts/quantize.py`。
- `src/view/InferencePage.h/.cpp`：新增“ONNX 量化模型优先使用 C++ Runtime”入口；不可用时明确提示并回退 Python 推理。
- `src/view/VisualizationPage.h/.cpp`：增加训练指标、模型文件、图像、类别、标注等数据库表展示。
- `scripts/train.py`：支持 `--train`、`--val`，训练后读取 `results.csv` 并输出 `FINAL_RESULT` JSON。
- `scripts/quantize.py`：新增统一量化入口，当前委托 `quantize_onnx.py` 完成 ONNX Runtime 动态 INT8 量化。

## 如何管理数据集

打开“数据集/统计”页：

- “创建数据集”：选择保存位置并输入名称，程序会创建 `train/images`、`train/labels`、`valid/images`、`valid/labels`、`test/images`、`test/labels` 和 `data.yaml`。
- “打开数据集”：选择已有 YOLO 数据集根目录。
- “导入图片文件夹”：把 jpg、jpeg、png、bmp、webp 图片复制到当前数据集的 `train/images`。
- 右侧表格会展示图片、分组、标注数量、类别 ID 和标签文件路径。

## 如何标注

打开“标注”页：

- 左侧选择图片文件夹并加载。
- 在画布空白处左键拖拽创建矩形框。
- 点击已有框选中，拖动框内部可移动，拖动四角白色控制点可改大小。
- 右键或中键拖动画布可平移，按钮支持放大、缩小、适应、左旋、右旋。
- “新增类别”“删除类别”维护类别列表。
- “应用到选中框”可把当前类别写入选中的矩形框。
- 标注表可双击编辑类别 ID、类别名和 YOLO 归一化坐标。
- “撤销”“重做”可回退或恢复画框、删框、移动、缩放、改类别等操作。
- “保存标签”会保存 YOLO txt，同时写入 SQLite 的 `images`、`categories`、`annotations` 表。

## 如何训练

打开“训练”页：

- 设置数据集根目录、训练集 images、验证集 images、模型/权重、epochs、batch size、image size、learning rate、device、输出目录、Python 和训练脚本。
- 点击“开始训练”，C++ 通过 `QProcess` 调用 `scripts/train.py`。
- Python stdout/stderr 会实时显示在日志窗口。
- 训练完成后，脚本输出 `BEST_MODEL:` 和 `FINAL_RESULT:`。
- GUI 会显示模型路径、save_dir、results.csv、results.png 以及最后一轮 metrics。
- 训练结果写入旧表 `train_records` 和新表 `training_runs`，其中 `training_runs.metrics_json` 保存结构化指标。

## 如何推理

打开“图像推理”页：

- 使用 QFileDialog 选择 jpg、jpeg、png、bmp、webp 图片。
- 选择 `.pt` 或 `.onnx` 模型，设置输出目录和 confidence。
- 默认通过 Python `scripts/infer_image.py` 加载模型并推理。
- 推理结果图显示在 GUI，检测类别、置信度、bbox 和 class id 显示在结果表。
- 结果写入 `inference_records`、`inference_objects` 和 `inference_results`。

## 如何量化

打开“模型部署/量化”页：

- 选择 Python 和 `.pt` 模型。
- 点击“导出 ONNX”，调用 `scripts/export_onnx.py`。
- 点击“动态量化 INT8”，调用 `scripts/quantize.py`，该脚本使用 ONNX Runtime 动态 INT8 PTQ。
- 成功后写入 `model_records` 和 `model_files`，包括量化模型路径、量化类型和后端。

## C++ ONNX Runtime 推理配置

当前项目提供可编译的条件编译入口 `QuantizedOnnxRunner`。默认未启用 ONNX Runtime C++，GUI 会提示并回退到 Python 推理。

如需启用 ORT C++，可在 `CarDet.pro` 中配置：

```qmake
DEFINES += CARDET_USE_ONNXRUNTIME
INCLUDEPATH += E:/path/to/onnxruntime/include
LIBS += -LE:/path/to/onnxruntime/lib -lonnxruntime
```

注意：当前 C++ ORT 类已经建立 session 接口和 GUI 降级提示，但完整 YOLO 预处理、NMS 后处理仍需根据模型输出格式补齐；未补齐前不应把该项当作满分功能。

## 数据库保存内容

- `dataset_stats`：数据集扫描统计。
- `datasets`：数据集目录和名称。
- `images`：图片路径、标签路径、标注数量、类别摘要。
- `categories`：类别 ID 和类别名。
- `annotations`：标注框类别和 YOLO 归一化坐标。
- `train_records`：兼容旧训练记录。
- `training_runs`：训练集/验证集路径、超参数、输出目录、模型路径、指标 JSON、状态、开始/结束时间。
- `model_records`：兼容旧模型记录。
- `model_files`：模型文件、量化类型、后端和创建时间。
- `inference_records`：兼容旧推理记录。
- `inference_objects`：每个检测对象。
- `inference_results`：推理结果摘要和后端。
