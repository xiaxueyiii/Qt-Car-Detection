# CarDet 项目模块说明文档

本文档用于说明 CarDet 车辆识别检测 Qt 桌面应用的主要模块、代码位置和功能分工，便于课程验收、GitHub 仓库展示和后续维护。

## 1. 项目简介

CarDet 是一个基于 Qt Widgets + C++ 开发的车辆识别/车辆检测桌面应用。项目围绕 YOLO 目标检测流程设计，提供数据集管理、图像标注、模型训练、ONNX 导出、INT8 动态量化、单图像推理、结果可视化和 SQLite 数据库存储等功能。

系统整体流程为：

数据集管理 -> 图像标注 -> YOLO 模型训练 -> ONNX 模型导出 -> INT8 动态量化 -> 图像推理 -> 结果展示 -> 数据库保存。

本项目不是单独的 Python 命令行检测脚本，而是通过 Qt/C++ 提供图形化操作界面，再通过 QProcess 调用 Python 深度学习脚本完成训练、推理、导出和量化任务。

## 3. 主窗口与界面模块

主窗口代码位于 `src/MainWindow.cpp` 和 `src/MainWindow.h`。该模块负责初始化数据库、创建主界面、添加功能标签页，并连接数据集页面、标注页面、训练页面、模型部署页面、推理页面和结果可视化页面。

主界面采用 Qt Widgets 实现，主要使用 `QMainWindow`、`QTabWidget`、`QMenuBar`、`QStatusBar`、`QPushButton`、`QTableWidget`、`QTextEdit` 等控件。用户可以通过图形化界面完成大部分操作，不需要直接输入复杂命令。

## 4. 数据集管理模块

相关代码：

- `src/view/DatasetPage.cpp/.h`
- `src/viewmodel/DatasetViewModel.cpp/.h`

主要功能：

- 选择 YOLO 数据集根目录。
- 扫描训练集、验证集和测试集中的图片与标签数量。
- 读取类别名称。
- 在界面中展示图片列表、标签状态和数据集统计信息。
- 支持创建或导入数据集，为后续标注和训练提供数据基础。

YOLO 数据集通常包含 `train/images`、`train/labels`、`valid/images`、`valid/labels`、`test/images`、`test/labels` 等目录。图片文件保存原始图像，标签文件保存每张图像对应的目标框信息。

## 5. 图像标注模块

相关代码：

- `src/view/AnnotationPage.cpp/.h`
- `src/view/AnnotationCanvas.cpp/.h`
- `src/viewmodel/AnnotationViewModel.cpp/.h`

主要功能：

- 加载数据集图片。
- 使用鼠标拖拽创建矩形标注框。
- 支持移动标注框、调整标注框大小、删除标注框。
- 支持新增类别、删除类别、修改标注框类别。
- 支持撤销和重做，降低误操作风险。
- 将标注结果保存为 YOLO 标签 txt 文件。
- 将图片、类别和标注信息写入 SQLite 数据库。

YOLO 标签格式为：

```text
class_id center_x center_y width height
```

其中 `center_x`、`center_y`、`width`、`height` 是相对于图片宽高的归一化坐标，取值一般在 0 到 1 之间。这样可以保证标签不依赖具体显示尺寸，即使图片在界面中被缩放，保存的标注仍然对应原始图像。

## 6. 模型训练模块

相关代码：

- `src/view/TrainingPage.cpp/.h`
- `src/viewmodel/TrainingViewModel.cpp/.h`
- `scripts/train.py`

主要功能：

- 在界面设置数据集路径、训练集路径、验证集路径。
- 在界面设置训练参数，例如模型权重、epochs、batch size、image size、learning rate、device 和输出目录。
- C++ 通过 `QProcess` 调用 Python 训练脚本。
- 实时读取 Python 标准输出和错误输出，并显示在训练日志窗口。
- 训练完成后显示模型文件路径和性能指标。
- 将训练参数、训练结果和模型文件记录保存到 SQLite 数据库。

训练脚本基于 Ultralytics YOLO 实现。训练完成后通常会生成 `best.pt` 和 `last.pt`，其中 `best.pt` 表示验证集指标最好的模型权重，常用于后续导出 ONNX 和图像推理。

## 7. 模型部署与量化模块

相关代码：

- `src/view/DeployPage.cpp/.h`
- `src/viewmodel/DeployViewModel.cpp/.h`
- `scripts/export_onnx.py`
- `scripts/quantize.py`
- `scripts/quantize_onnx.py`

主要功能：

- 选择训练得到的 `.pt` 模型权重。
- 将 PyTorch YOLO 模型导出为 `.onnx` 模型。
- 对 ONNX 模型进行 INT8 动态量化。
- 在界面显示导出和量化日志。
- 保存模型文件信息到数据库。

ONNX 是一种通用模型交换格式，便于模型部署到不同推理框架中。INT8 动态量化可以减小模型体积，并在部分硬件和推理框架中提升运行效率，但也可能带来一定精度损失。

## 8. 图像推理模块

相关代码：

- `src/view/InferencePage.cpp/.h`
- `src/viewmodel/InferenceViewModel.cpp/.h`
- `scripts/infer_image.py`
- `src/model/QuantizedOnnxRunner.cpp/.h`

主要功能：

- 通过 GUI 文件对话框选择待检测图片。
- 选择 `.pt`、`.onnx` 或 INT8 `.onnx` 模型。
- 设置置信度阈值 `conf`。
- 使用 Python YOLO 或 ONNX Runtime 进行单图像推理。
- 在界面显示检测结果图。
- 在表格中显示类别、置信度、边界框和类别编号。
- 将推理记录和检测目标写入 SQLite 数据库。

项目同时提供 Python 推理路径和 C++ ONNX Runtime 推理路径。Python 推理调用简单，适合快速验证；C++ ONNX Runtime 推理更符合桌面端部署要求，适合展示 C++ 加载量化模型进行推理的能力。

## 9. 结果可视化模块

相关代码：

- `src/view/VisualizationPage.cpp/.h`

主要功能：

- 读取数据库中的训练记录、模型记录和推理记录。
- 在界面中展示训练结果、模型文件和推理结果。
- 辅助课程验收时证明系统不是只显示临时日志，而是能够将关键结果结构化保存。

## 10. 数据库模块

相关代码：

- `src/model/DatabaseManager.cpp/.h`
- `src/model/DataTypes.h`

主要功能：

- 初始化 SQLite 数据库。
- 创建数据集、图片、类别、标注、训练记录、模型文件、推理记录等数据表。
- 保存标注框信息。
- 保存训练参数、训练时间、指标和模型路径。
- 保存推理图片、模型路径、结果图、JSON 文件和检测目标。
- 提供结果查询接口，供可视化页面展示。

SQLite 是轻量级本地数据库，不需要单独安装数据库服务器，适合课程项目和桌面应用使用。

## 11. Python 算法脚本模块

相关脚本：

- `scripts/train.py`：训练 YOLO 模型。
- `scripts/infer_image.py`：单图像推理，输出结果图和 JSON。
- `scripts/export_onnx.py`：将 `.pt` 模型导出为 ONNX。
- `scripts/quantize.py`：量化入口脚本。
- `scripts/quantize_onnx.py`：使用 ONNX Runtime 完成动态量化。
- `scripts/performance_benchmark.py`：性能测试脚本。

Qt/C++ 端通过 `QProcess` 调用这些脚本，并实时接收脚本输出。这样设计的好处是界面部分保持 C++ 桌面应用形态，算法部分可以复用成熟的 Python 深度学习生态。

## 12. 主要技术栈

- 桌面界面：Qt Widgets
- 编程语言：C++20、Python
- 构建方式：qmake
- 目标检测框架：Ultralytics YOLO
- 模型格式：PyTorch `.pt`、ONNX `.onnx`
- 量化方式：ONNX Runtime 动态量化
- 推理方式：Python YOLO、Python ONNX Runtime、C++ ONNX Runtime
- 数据库存储：SQLite

## 13. 运行和演示建议

课程验收时建议按以下顺序展示：

1. 打开 Qt 主界面，说明这是 Qt/C++ 桌面应用。
2. 展示数据集页面，说明数据集图片、标签和类别统计。
3. 展示标注页面，演示矩形框创建、移动、删除、类别修改和保存。
4. 展示训练页面，说明训练参数，并展示训练日志实时输出。
5. 展示模型部署页面，演示 `.pt` 导出 ONNX 和 INT8 动态量化。
6. 展示图像推理页面，选择图片和模型，运行检测并显示结果图。
7. 展示结果可视化页面或数据库记录，说明训练和推理结果已经保存。
